# 📚 밀리의 서재 AI 검색 서비스 프로토타입

## 프로젝트 개요

실제 백엔드 없이도 로직이 완벽하게 시뮬레이션되는 **밀리의 서재 AI 검색 서비스** React 기반 웹 프로토타입입니다.

## 주요 기능

### 🎯 3가지 검색 시나리오

#### Case A: BM25 검색 (키워드 기반)
- **예시**: "트렌드 코리아 2025"
- 키워드 분석 후 관련 도서 리스트 즉시 노출
- 기본적인 검색 엔진 방식

#### Case B: RAG 추천 (감정/의도 기반)
- **예시**: "우울할 때 읽기 좋은 에세이 추천해줘"
- 의도 분석 UI 표시
- AI 스트리밍 답변 (타이핑 효과)
- 추천 이유가 포함된 도서 카드 3종
- 꼬리물기 질문 제공

#### Case C: Failover (대체 큐레이션)
- **예시**: "희귀본 검색"
- 외부 DB 탐색 UI 표시
- 비슷한 주제의 대체 도서 추천

## 핵심 UX 특징

### ✨ 로딩 메시지
검색 중 다양한 감성적인 메시지가 랜덤하게 표시됩니다:
- "작은 AI 요정들이 도서관을 헤집는 중..."
- "질문의 행간을 읽고 있어요..."
- "당신을 위한 완벽한 책을 찾고 있어요..."

### 📖 도서 카드 컴포넌트
- 책 표지 (색상 플레이스홀더)
- 도서명, 저자
- 요약 정보
- AI 추천 이유 (RAG 모드)
- '내 서재 담기' 버튼

### 💬 AI 스트리밍 효과
RAG 모드에서 AI 답변이 타이핑하듯 한 글자씩 생성되는 리얼리스틱한 효과

### 🔄 꼬리물기 질문
검색 결과 하단에 관련 질문 칩 버튼 제공:
- "더 위로가 되는 책 추천해줘"
- "이 작가의 다른 책은?"
- "감동적인 소설 알려줘"

### 📜 대화 기록 사이드바
- 햄버거 메뉴를 통해 이전 검색 기록 확인
- 클릭 시 해당 검색 결과로 복원

## Mock 데이터

### 검색 결과 데이터
- 트렌드 코리아 시리즈
- 90년생이 온다
- 요즘 어른들의 사회

### RAG 추천 데이터
- **우울/힐링**: 미드나잇 라이브러리, 달러구트 꿈 백화점, 언어의 온도
- **힐링**: 나는 나로 살기로 했다, 보노보노처럼 살다니 다행이야

### Failover 데이터
- 이처럼 사소한 것들
- 불편한 편의점
- 작별하지 않는다

## 로직 시뮬레이션

### 타이밍 (setTimeout)
- **BM25**: 1.5초 로딩
- **RAG**:
  - 0.8초 (의도 분석)
  - 1.2초 (AI 처리)
  - 스트리밍 효과 (30ms/글자)
- **Failover**:
  - 1초 (내부 검색)
  - 1.5초 (외부 탐색)

### 검색 타입 감지
```javascript
function detectSearchType(query) {
  // "추천", "우울", "힐링" 등 → RAG
  // "희귀", "절판" 등 → FAILOVER
  // 기타 → BM25
}
```
